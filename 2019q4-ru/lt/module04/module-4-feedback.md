# SQL Mentoring module 4 feedback

# Общее

1. Coding style. В программе существует правила, соглашения которым 
необходимо следовать при выполнении заданий. Описание находится в 
файле "Результат выполнения.docx"
2. Лучше не устанавливать IDENTITY_INSERT. Удобство idenity 
полей как раз в том, чтобы явно не указывать значения при вставке,
т.к. они будут автоматически вставлены уникальным образом.
3. Формат дат в строках. Давай придерживаться стандарта ANSI, поэтому
даты лучше писать в формате 'yyyyMMdd'

литералы начиная с N -> N'GODOS', N'Tarte au sucre'
